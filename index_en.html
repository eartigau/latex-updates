Installation

After installation, the latex_updates command is available everywhere in your terminal — no need to be in the script's directory.

1 — Install the package

# Directly from GitHub (recommended)
$ pip install git+https://github.com/eartigau/latex-updates.git

# Or clone and install locally
$ git clone https://github.com/eartigau/latex-updates.git
$ pip install ./latex-updates

2 — Install latexdiff

# macOS (Homebrew)
$ brew install latexdiff

# Debian / Ubuntu
$ sudo apt install latexdiff

# TeX Live (any platform)
$ tlmgr install latexdiff

pdflatex is needed only for the automatic PDF compilation step; if it is not installed, latex_updates still produces the .tex file and tells you how to compile it manually. No Python dependencies beyond the standard library are required.

Quick start

Try the included demo manuscripts (clone the repo first to get them):

$ latex_updates demo/demo_old.tex demo/demo_new.tex

latex_updates: generating diff between
  old: demo/demo_old.tex
  new: demo/demo_new.tex

  Cleaning demo_old.tex (strip {\bf} markup)...
  Cleaning demo_new.tex (strip bare groups)...
  Running latexdiff...
  Inserting colour definitions and toggle...

  Diff source → demo/diff_demo_new.tex
  Compiling PDF (2 passes)...
  Diff PDF    → demo/diff_demo_new.pdf

Done.

Open demo/diff_demo_new.pdf to see the result.

Usage

latex_updates OLD.tex NEW.tex [options]
Argument / optionDescription
OLD.tex Previous version of the manuscript
NEW.tex New (revised) version of the manuscript
-o FILE / --output FILE Override the output filename.
Default: diff_<NEW>.tex in the same directory
--no-compile Skip pdflatex; produce .tex only

Examples

# Basic — produces diff_v2_paper.tex and diff_v2_paper.pdf
$ latex_updates v1_paper.tex v2_paper.tex

# Custom output name (handy for sending to a referee)
$ latex_updates old.tex new.tex -o for_referee.tex

# Generate .tex only, compile later
$ latex_updates old.tex new.tex --no-compile
$ pdflatex diff_new.tex

Showing or hiding deleted text

Every generated .tex file contains a toggle near the top that lets you switch between two modes without re-running the script:

Show deleted text (default — for the referee response)

\newif\ifshowdel
\showdeltrue   % ← deleted text in orange

Hide deleted text (clean copy for resubmission)

\newif\ifshowdel
\showdelfalse  % ← deleted text invisible

Change the line in the .tex file and recompile — no other edits needed.

How it works

1

Clean the old file

Strip {\bf …} and \textbf{…} markup added during earlier revision rounds.

2

Clean the new file

Remove leftover bare { } groups from the {\bf} → { } conversion process.

3

Run latexdiff

Word-level diff with math-mode protection. The latexdiff preamble is kept intact for safety.

4

Inject colour

Blue for additions, orange with strikethrough for deletions. The toggle is injected before \begin{document}.

5

Compile PDF

Two pdflatex passes resolve cross-references and produce the final PDF.

Colour reference

The occurrence rate was previously $f = 1.4^{+0.6}_{-0.4}$\,\% $f = 1.07^{+0.31}_{-0.24}$\,\%, revised downward with the expanded 2 126-star sample and improved completeness corrections.

Works with any LaTeX class

Tested with article, aa (A&A journal class), aastex63 (AAS journals), and mnras. Does not alter any class-specific commands.

Demo manuscripts

The demo/ folder contains two realistic astronomy papers that illustrate typical first-round and second-round submissions.

FileContent
demo/demo_old.tex First submission: 1 800-star sample, 14 recovered companions, occurrence rate 1.4%, separation slope β = −0.9
demo/demo_new.tex Revised after referee: 2 126 stars, 17 companions + 2 new candidates, rate 1.07%, β = −1.38, new Discussion section

The changes span: updated numbers in the abstract, a new introduction paragraph discussing formation models, an additional row in the results table, a new MCMC results table, and a full new Discussion section — giving a representative range of diff types.
